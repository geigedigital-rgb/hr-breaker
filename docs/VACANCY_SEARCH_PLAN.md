# План: «Найти вакансию» — поиск актуальных вакансий

## Цель
Реализовать блок поиска вакансий с актуальными объявлениями из популярных сервисов (для Германии и при необходимости других регионов): типичный поиск, лёгкий раскрывающийся фильтр, бейджи (популярные запросы + ранее введённые пользователем), карточки вакансий в стиле приложения.

---

## 1. Источники данных (вакансии)

| Источник | Регион | Доступ | Примечание |
|----------|--------|--------|------------|
| **Adzuna API** | DE, AT, CH, и др. | Бесплатно (app_id + app_key) | JSON, фильтры: what, where, full_time, permanent, salary_min, sort_by. Идеально для старта. |
| StepStone | DE | Платно (Apify / JobFeed) | Подключить позже при необходимости. |
| Indeed | Много стран | Платно / парсинг | Опционально, фаза 2. |

**Рекомендация для первой версии:** один источник — Adzuna (Германия, `country=de`). Регистрация: https://developer.adzuna.com/ → получение `app_id` и `app_key`, хранить в `.env`.

---

## 2. Структура данных

### 2.1 Модель вакансии (карточка, как в сервисах поиска работы)

Унифицированный формат на фронте и в ответе бэкенда:

```ts
// frontend: api.ts или types
type VacancyCard = {
  id: string;              // уникальный (например provider + external_id)
  title: string;           // должность
  company: string;         // компания
  location: string | null; // город/регион
  salary_min?: number | null;
  salary_max?: number | null;
  salary_text?: string | null;  // "от 50 000 €" или "по договорённости"
  contract_type?: string | null; // "Full-time", "Part-time", "Permanent"…
  posted_at?: string | null;     // ISO дата публикации
  snippet?: string | null;       // краткое описание (1–2 предложения)
  url: string;                  // ссылка на вакансию на источнике
  source: string;               // "adzuna" | "stepstone" | …
};
```

Маппинг ответа Adzuna → `VacancyCard` делается на бэкенде (или во фронте, если запрос с фронта через наш API).

### 2.2 Ответ поиска

```ts
type VacancySearchResponse = {
  items: VacancyCard[];
  total: number;
  page: number;
  page_size: number;
};
```

### 2.3 Запрос поиска (фильтры)

```ts
type VacancySearchParams = {
  q: string;           // поисковая строка (должность/ключевые слова)
  location?: string;   // город/регион
  full_time?: boolean;
  permanent?: boolean;
  salary_min?: number;
  page?: number;
  page_size?: number;
};
```

## 3. Бэкенд (пошагово)

### Шаг 3.1 Конфиг и env
- Добавить в `.env.example`: `ADZUNA_APP_ID=`, `ADZUNA_APP_KEY=`.
- В `config.py` (или аналоге) читать эти переменные; при отсутствии — эндпоинт поиска возвращает 503 или пустой результат с сообщением «поиск не настроен».

**Проверка:** приложение стартует без падения при отсутствии ключей; при наличии ключей запрос к Adzuna выполняется.

### Шаг 3.2 Эндпоинт прокси поиска
- Маршрут: `GET /api/vacancies/search` (или `GET /api/jobs/search`).
- Параметры: `q`, `location`, `full_time`, `permanent`, `salary_min`, `page`, `page_size`.
- Логика:
  - Валидация параметров (q не пустой, page/page_size в разумных пределах).
  - Запрос к Adzuna: `https://api.adzuna.com/v1/api/jobs/de/search/{page}?app_id=...&app_key=...&what=...&where=...` и др.
  - Маппинг JSON Adzuna → список `VacancyCard`, возврат `VacancySearchResponse`.

**Проверка:** curl/Postman с `q=developer` возвращает JSON с полями `items`, `total`, `page`, `page_size`; карточки содержат title, company, location, url.

### Шаг 3.3 (Опционально) История поисков пользователя
- Таблица или ключ по `user_id`: последние N запросов (например, только `q` и `location`).
- Эндпоинт: `GET /api/vacancies/search-history` — возвращает массив строк/объектов для бейджей «ранее введённые».
- При успешном поиске с авторизованным пользователем — сохранять запрос в историю.

**Проверка:** после нескольких поисков бейджи «ранее введённые» показывают последние запросы.

## 4. Фронтенд (пошагово)

### Шаг 4.1 Маршрут и страница
- Вариант A: отдельная страница `/vacancies` (или `/jobs`), переход по клику «Найти вакансию» с Главной.
- Вариант B: тот же экран Главной, но при клике «Найти вакансию» показывать не скелетон, а полноценный блок поиска (как сейчас скелетон, но с реальным UI).

Рекомендация: **отдельная страница** `/vacancies` — проще верстка, URL можно шарить. В `App.tsx` добавить `<Route path="vacancies" element={<Vacancies />} />`. Кнопка «Найти вакансию» на Главной — `<Link to="/vacancies">` или `navigate('/vacancies')`.

**Проверка:** с Главной переход на `/vacancies`, отображается страница поиска.

### Шаг 4.2 Верхний блок: поиск + раскрывающийся фильтр
- Строка поиска: одно поле (как в сервисах) — «Должность или ключевые слова», обязательное.
- Кнопка «Найти» или поиск по Enter.
- Рядом кнопка «Фильтры» (раскрывающийся блок):
  - Место (город/регион) — опционально.
  - Тип занятости: Full-time / Part-time / и т.д. (чекбоксы или селект).
  - Оклад от (число) — опционально.
  - «Постоянный контракт» — чекбокс (соответствует Adzuna `permanent`).
- Стиль: только палитра приложения — фон `#FAFAFC`/`#F2F3F9`, бордер `#EBEDF5`, акцент `#4578FC`, текст `#181819` / `var(--text-muted)`.

**Проверка:** ввод запроса и применение фильтров формируют запрос к `/api/vacancies/search` с нужными query-параметрами.

### Шаг 4.3 Бейджи
- Два ряда (или один с разделением):
  - **Популярные:** захардкоженный список (например: «Python Developer», «Backend», «Data Analyst», «Frontend», «DevOps», «Product Manager») — клик подставляют в поле поиска и запускают поиск.
  - **Ранее введённые:** из `GET /api/vacancies/search-history` (если реализовано); иначе пока пусто. Клик — подставить в поиск и запустить.
- Визуально: бейджи в стиле приложения (скругление, нейтральный фон, при hover — лёгкий акцент).

**Проверка:** клик по бейджу заполняет поиск и уходит запрос; бейджи не ломают вёрстку на мобиле.

### Шаг 4.4 Список результатов и карточки
- Список: вертикальный скролл, пагинация (страницы или «Ещё»).
- Карточка вакансии (типичная информация, как в сервисах):
  - Заголовок (должность) — крупно, кликабельный (ссылка на `url`).
  - Компания.
  - Локация (если есть).
  - Зарплата (если есть) — одной строкой.
  - Тип контракта (если есть).
  - Дата публикации (если есть) — кратко («2 дня назад»).
  - Snippet (1–2 строки).
- Стиль строго приложения: белый/светлый фон карточки, бордер `#EBEDF5`, без пёстрых цветов; акцент только для ссылки и кнопок.

**Проверка:** после поиска отображаются карточки с полями из модели; клик по заголовку открывает вакансию на источнике в новой вкладке.

### Шаг 4.5 Состояния и ошибки
- Загрузка: скелетоны карточек или спиннер.
- Пустой результат: «По запросу ничего не найдено. Измените параметры».
- Ошибка сети/5xx: «Не удалось загрузить вакансии. Попробуйте позже.»

**Проверка:** при отключённом бэкенде/неверных ключах показывается сообщение об ошибке, не белый экран.

## 5. Стиль (строго наше приложение)

- Фон страницы: `#F2F3F9`.
- Карточки/панели: `bg-white`, `border-[#EBEDF5]`, `rounded-2xl` / `rounded-xl`.
- Текст: заголовки `#181819`, вторичный `text-[var(--text-muted)]`.
- Кнопки/ссылки: акцент `#4578FC`, hover темнее.
- Инпуты: бордер `#EBEDF5`, focus ring `#4578FC`.
- Бейджи: нейтральный фон `#EBEDF5` или `#F0F2F8`, при выборе — лёгкий фиолетовый `#4578FC`/12.
- Не использовать: ярко-зелёный, оранжевый, розовый для фонов карточек; только серые и один синий акцент.
